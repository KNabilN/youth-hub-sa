
# تحويل صفحة إدارة الشكاوى إلى شكل جدول (مثل الخدمات وطلبات الجمعيات)

## الهدف
تحويل صفحة `AdminDisputes.tsx` من عرض البطاقات (Cards) إلى عرض جدول (Table) ليتطابق مع تصميم صفحتي إدارة الخدمات وطلبات الجمعيات.

## التغييرات المطلوبة

### تعديل `src/pages/admin/AdminDisputes.tsx`
- استبدال عرض البطاقات بجدول بيانات يحتوي على الأعمدة التالية:
  - المشروع (عنوان المشروع كرابط لصفحة التفاصيل)
  - مقدم الشكوى
  - الحالة (Badge ملونة)
  - التاريخ
  - تغيير الحالة (Select لتغيير الحالة مباشرة)
  - إجراءات (زر عرض + زر تعديل)
- إضافة حقل بحث بالنص
- إضافة ترقيم الصفحات (PaginationControls)
- إضافة حوار التعديل المباشر (AdminDirectEditDialog) لتعديل ملاحظات الحل
- توحيد تصميم الهيدر مع الصفحات الأخرى (عنوان بسيط بدون أيقونة كبيرة)
- استخدام نفس نمط الفلاتر (بحث + حالة + زر إعادة تعيين)

### التفاصيل التقنية
- استيراد مكونات الجدول: `Table, TableBody, TableCell, TableHead, TableHeader, TableRow`
- استخدام `usePagination` و `PaginationControls` للترقيم
- استخدام `useUpdateDispute` لتغيير الحالة مباشرة من الجدول
- استخدام `disputeStatusLabels` و `disputeStatusColors` للألوان والتسميات
- استخدام `format` من `date-fns` لعرض التاريخ بنفس التنسيق
- إضافة أعمدة: العنوان (المشروع)، مقدم الشكوى، الحالة، التاريخ، تغيير الحالة، إجراءات (عرض)
