

## إنشاء عقد احترافي قابل للمراجعة والتعديل بعد الدفع

### المشكلة الحالية
عند الدفع (سواء شراء خدمة أو قبول عرض)، يتم إنشاء العقد مع توقيع الجمعية تلقائياً (`association_signed_at = now()`) بدون أن تراجعه الجمعية. العقد يحتوي فقط على جملة واحدة مثل `عقد تنفيذ خدمة "X" بقيمة Y ر.س` بدون أي بنود قانونية.

### الحل
1. **إنشاء العقد بدون توقيع** — بعد الدفع يُنشأ العقد بشروط تفصيلية لكن بدون `association_signed_at`
2. **توجيه الجمعية لصفحة العقد** — بعد الدفع الناجح يتم التوجيه لصفحة تفاصيل الطلب تبويب العقد
3. **شاشة مراجعة العقد** — عرض العقد بتصميم احترافي يتضمن:
   - بنود المنصة الثابتة (حل النزاعات طبقاً للقانون السعودي، حماية الأطراف)
   - نطاق العمل (مأخوذ من وصف الخدمة/الطلب) — **قابل للتعديل** من الجمعية قبل التوقيع
   - تفاصيل مالية (المبلغ، مدة التنفيذ)
4. **الجمعية تعدل نطاق العمل وتوقّع** → يصل إشعار للمزود
5. **المزود يراجع ويوقّع** → يبدأ المشروع

### التغييرات المطلوبة

#### 1. مكون جديد: `ContractReviewPanel`
ملف: `src/components/contracts/ContractReviewPanel.tsx`

مكون يعرض العقد بشكل احترافي مقسم لأقسام:
- **ترويسة**: رقم العقد، التاريخ، أطراف العقد
- **البنود الثابتة**: 4-5 بنود قانونية (حل النزاعات عبر المنصة طبقاً لأنظمة المملكة العربية السعودية، السرية، حقوق الملكية الفكرية، إلخ)
- **نطاق العمل**: textarea قابل للتعديل من الجمعية فقط قبل التوقيع
- **التفاصيل المالية**: المبلغ، مدة التنفيذ
- **زر التوقيع** مع تأكيد

#### 2. تعديل إنشاء العقود (إزالة التوقيع التلقائي)
الملفات المتأثرة:
- `supabase/functions/moyasar-verify-payment/index.ts` — في `processProjectPayment` و `processCheckout`: إزالة `association_signed_at` + إضافة شروط عقد تفصيلية
- `src/hooks/usePurchaseService.ts` — نفس التعديل
- `src/pages/Checkout.tsx` — مسار `grant_balance`: نفس التعديل
- `src/components/bids/BidPaymentDialog.tsx` — مسار `grant_balance`: نفس التعديل

بدلاً من `terms: 'عقد تنفيذ...'` سيتم إنشاء terms JSON مهيكل أو نص غني يحتوي:
```
بنود العقد:
1. نطاق العمل: [وصف الخدمة/الطلب]
2. القيمة المالية: [المبلغ] ر.س
3. تلتزم المنصة بالفصل في أي نزاع طبقاً لأنظمة المملكة العربية السعودية
4. يحتفظ كل طرف بسرية المعلومات...
5. تنتقل حقوق الملكية الفكرية للجمعية بعد إتمام العمل...
```

#### 3. تعديل `useSignContract` 
ملف: `src/hooks/useContracts.ts`

إضافة mutation جديد `useUpdateContractTerms` يسمح بتعديل `terms` قبل التوقيع (الجمعية فقط إذا لم توقّع بعد).

#### 4. تعديل عرض العقد في `ProjectDetails.tsx`
استبدال العرض البسيط الحالي بمكون `ContractReviewPanel` الجديد.

#### 5. تعديل التوجيه بعد الدفع
- `src/pages/PaymentCallback.tsx` / `PaymentSuccess.tsx` — إضافة زر "مراجعة العقد" يوجه لصفحة الطلب
- إظهار بانر تنبيهي في الصفحة الرئيسية/الطلبات عند وجود عقود تنتظر توقيع

#### 6. تعديل `ContractCard.tsx`
تحديث بطاقة العقد في صفحة العقود لعرض ملخص البنود بشكل أفضل.

### البنود القانونية الثابتة (تُضاف لكل عقد)
1. تدار هذه الاتفاقية وتُفسر وفقاً لأنظمة المملكة العربية السعودية
2. تتولى المنصة الفصل في أي نزاع ينشأ بين الأطراف
3. يلتزم الطرفان بالحفاظ على سرية جميع المعلومات المتبادلة
4. تنتقل حقوق الملكية الفكرية للأعمال المنجزة إلى الطرف الأول (الجمعية) بعد إتمام الدفع
5. يحق لأي طرف إلغاء العقد قبل بدء التنفيذ مع استرداد كامل المبلغ
