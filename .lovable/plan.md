

## خطة: تحسين إجراءات الضمان المالي في صفحة تفاصيل الشكوى

### المشكلة الحالية

صفحة `AdminDisputeDetail.tsx` تنفذ تحديث الضمان المالي (تحرير/استرداد) **بشكل مباشر** دون:
1. إلزام الأدمن برفع إيصال تحويل بنكي
2. إصدار فاتورة تلقائية
3. إخفاء الأزرار بعد تنفيذ الإجراء (يمكن الضغط مرة أخرى)
4. إرسال إشعارات (الإشعارات تُرسل بالفعل عبر trigger `notify_on_escrow_change` لكن الفاتورة لا تُصدر)

بينما صفحة `AdminFinance.tsx` لديها بالفعل **النمط الصحيح**: dialog لرفع إيصال → optimistic lock → رفع الملف → تحديث الضمان → إصدار فاتورة.

### الحل

**ملف واحد يتغير: `src/pages/admin/AdminDisputeDetail.tsx`**

1. **إضافة استعلام لجلب بيانات الضمان الحالية** للمشروع المرتبط بالشكوى (الحالة، المبلغ، payer_id, payee_id) — لتحديد الأزرار المتاحة وإخفائها عند عدم الحاجة

2. **إضافة Dialog لرفع الإيصال** عند الضغط على "تحرير" أو "استرداد":
   - نفس نمط `AdminFinance.tsx` (handleEscrowWithReceipt)
   - رفع الإيصال لـ `escrow-receipts` bucket
   - تحديث الضمان بـ optimistic lock (`.eq("status", "held")` أو `"frozen"`)
   - إصدار فاتورة تلقائية (invoice) بالعمولة المحسوبة
   - الإشعارات تُرسل تلقائياً عبر trigger الموجود

3. **إخفاء الأزرار حسب حالة الضمان الفعلية**:
   - إذا الضمان `released` أو `refunded`: إخفاء أزرار التحرير والاسترداد
   - إذا الضمان `frozen`: إظهار أزرار التحرير والاسترداد وإلغاء التجميد فقط
   - إذا الضمان `held`: إظهار كل الأزرار ما عدا إلغاء التجميد
   - إذا لا يوجد ضمان: إخفاء القسم بالكامل

4. **تجميد/إلغاء التجميد** يبقى بدون إيصال (لأنه لا يتضمن تحويل أموال)
