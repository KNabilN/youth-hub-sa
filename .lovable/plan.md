

## المشكلة

1. **جدول المنح المستلمة** لا يوضح نوع المنحة (عامة أم مخصصة لطلب/خدمة معينة)
2. **منح مخصصة لمشروع** يمكن استخدامها في مشاريع أخرى - لا يوجد تقييد
3. **لا يوجد UX للدفع المختلط** - إذا لم تغطِ المنح كامل التكلفة، لا يستطيع المستخدم إكمال الفرق بطريقة دفع أخرى

## الحل

### 1. تحسين جدول المنح المستلمة (`src/pages/ReceivedGrants.tsx`)

- إضافة عمود "النوع" يوضح: **دعم عام** (project_id و service_id فارغين) أو **مخصص لطلب** (مع اسم الطلب) أو **مخصص لخدمة**
- إضافة عمود يعرض اسم المشروع/الخدمة المرتبطة
- استخدام badges ملونة للتفريق بين الأنواع

### 2. تقييد استخدام المنح المخصصة (`src/hooks/usePayFromGrants.ts`)

- عند الدفع لمشروع معين، جلب المنح المتاحة على مرحلتين:
  1. أولاً: المنح المخصصة لنفس المشروع (`project_id = projectId`)
  2. ثانياً: المنح العامة فقط (`project_id IS NULL AND service_id IS NULL`)
- المنح المخصصة لمشروع آخر لا تُستخدم أبداً
- نفس المنطق للخدمات

### 3. دعم الدفع المختلط في `BidPaymentDialog`

- تغيير منطق `hasGrantBalance` ليحسب الرصيد المتاح للمشروع المحدد (مخصص + عام)
- إذا كان الرصيد يغطي جزءاً فقط: عرض خيار "دفع مختلط" يوضح:
  - المبلغ من رصيد المنح (المخصص + العام)
  - المبلغ المتبقي المطلوب دفعه إلكترونياً أو بنكياً
- UI يعرض شريط تقدم أو بطاقة توضح التوزيع

```text
┌─────────────────────────────────────────┐
│ رصيد المنح المتاح لهذا الطلب            │
│                                          │
│ منح مخصصة للطلب:     3,000 ر.س          │
│ رصيد المنح العام:    2,000 ر.س          │
│ ─────────────────────────────           │
│ الإجمالي المتاح:     5,000 ر.س          │
│                                          │
│ تكلفة الطلب:        8,000 ر.س           │
│ المتبقي للدفع:      3,000 ر.س           │
│                                          │
│ ○ دفع المتبقي إلكترونياً               │
│ ○ دفع المتبقي بتحويل بنكي              │
└─────────────────────────────────────────┘
```

### 4. Hook جديد لحساب الرصيد المتاح لمشروع محدد

إنشاء `useProjectGrantBalance` في `src/hooks/useAssociationGrants.ts`:
- يجلب المنح المتاحة المخصصة للمشروع + المنح العامة
- يرجع `{ projectSpecific, general, total }`

| الملف | التعديل |

| `src/components/bids/BidPaymentDialog.tsx` | دعم الدفع المختلط (منح + إلكتروني/بنكي) مع UX واضح |
