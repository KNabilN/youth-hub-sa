

# مراجعة شاملة للنظام قبل الإطلاق

## الحالة العامة
الموقع منشور بالفعل على `youth-hub-sa.lovable.app` مع رؤية عامة (public). البنية العامة سليمة: الصفحة الرئيسية تعمل، التنقل واضح، التصميم متسق. لكن هناك مشاكل أمنية وتحسينات مطلوبة قبل الاستخدام الفعلي.

---

## 1. مشاكل أمنية حرجة (يجب إصلاحها فوراً)

### أ. بيانات بنكية مكشوفة للجميع
جدول `profiles` يحتوي على بيانات حساسة (IBAN، رقم الحساب، أرقام هواتف) وسياسة RLS تسمح لأي مستخدم مسجّل بقراءة كل الملفات الشخصية بالكامل.

**الحل**: إنشاء سياسة RLS جديدة تسمح للمستخدم بقراءة ملفه فقط بالكامل، وإنشاء دالة `get_public_profile` محدودة للآخرين (موجودة بالفعل لكن الاستعلامات المباشرة لا تزال تكشف البيانات).

### ب. أي مستخدم يمكنه إرسال إشعارات لأي مستخدم آخر
سياسة INSERT على `notifications` تشترط فقط `auth.uid() IS NOT NULL` مما يسمح بإرسال إشعارات مزيفة.

**الحل**: تغيير السياسة لتقييد الإدراج للـ triggers فقط (SECURITY DEFINER) أو إزالة سياسة INSERT العامة.

### ج. كشف أدوار المستخدمين
سياسة `user_roles` تسمح لأي مستخدم بمعرفة من هو الـ admin.

**الحل**: إبقاء سياسة "Users can view own role" فقط وإزالة "Authenticated users can view all roles" أو تقييدها.

### د. حماية كلمات المرور المسربة معطّلة
**الحل**: تفعيل Leaked Password Protection.

## 3. تحسينات الواجهة والتصميم

### أ. الصفحة الرئيسية
- **اسم المنصة في الهيدر**: النص "منصة الخدمات المشتركـــة" يبدو ممدوداً بشكل غريب — يحتاج ضبط
- **قسم الخدمات**: زر "أضف للسلة" يظهر للزوار غير المسجلين ويستدعي `guardAction` الذي يتطلب `useProfile` — قد يسبب خطأ للزائر غير المسجّل. يجب فحص هذا السيناريو وإظهار زر "سجّل الدخول" بدلاً منه

### ب. نص الهيدر
- `site_name` يأتي من `useSiteContent("header")` — إذا كان فارغاً يستخدم "الخدمات المشتركة" كنص بديل، وهذا مناسب

### ج. تناسق الألوان
- التصميم متسق (أخضر/أزرق) مع دعم الوضع الداكن — جيد

---

## 4. قائمة التغييرات المطلوبة

### تغييرات قاعدة البيانات (migrations):
1. تعديل RLS على `profiles`: تقييد القراءة الكاملة للمالك فقط، وإنشاء سياسة محدودة للآخرين
2. تعديل RLS على `notifications`: إزالة سياسة INSERT العامة (الإشعارات تُنشأ عبر triggers بالفعل)
3. تعديل RLS على `user_roles`: إزالة سياسة "Authenticated users can view all roles"
4. تعديل RLS على `profile_saves`: تقييد SELECT للمالك فقط

### تغييرات في الكود:
5. `LandingServicesGrid.tsx`: فحص حالة المستخدم غير المسجّل قبل استدعاء `guardAction` — إظهار زر "سجّل الدخول" بدلاً من "أضف للسلة" للزوار
6. تفعيل Leaked Password Protection عبر أدوات الإعدادات

- منع `super_admin` عند التسجيل ✅
- تحقق التوثيق (`useVerificationGuard`) ✅

- staleTime مناسب للاستعلامات ✅
