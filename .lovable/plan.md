
# إضافة صفحة تفاصيل الشكوى للأدمن

## الهدف
السماح للأدمن بالنقر على أي شكوى في صفحة إدارة الشكاوى لفتحها في صفحة مستقلة تعرض كافة التفاصيل.

## التغييرات المطلوبة

### 1. إنشاء صفحة `src/pages/admin/AdminDisputeDetail.tsx`
صفحة مستقلة داخل `DashboardLayout` تعرض:
- عنوان المشروع المرتبط بالشكوى مع رابط لصفحة تفاصيل المشروع
- اسم مقدم الشكوى وتاريخ الإنشاء
- حالة الشكوى مع إمكانية تغييرها (نفس منطق DisputeCard)
- وصف الشكوى كاملاً
- الأثر المالي (DisputeFinancialImpact)
- إجراءات الضمان المالي (تحرير، استرداد، تجميد، إلغاء التجميد)
- الجدول الزمني للشكوى (DisputeTimeline)
- سجل المحادثات (DisputeResponseThread)
- ملاحظات الحل
- زر العودة للقائمة

### 2. تعديل `src/App.tsx`
- إضافة استيراد lazy للصفحة الجديدة
- إضافة مسار `/admin/disputes/:id` داخل `AdminRoute`

### 3. تعديل `src/pages/admin/AdminDisputes.tsx`
- جعل كل بطاقة شكوى قابلة للنقر عبر تغليفها برابط `Link` يوجه إلى `/admin/disputes/{id}`

### 4. تعديل `src/components/admin/DisputeCard.tsx`
- إضافة خاصية اختيارية `linkTo` أو جعل عنوان البطاقة رابطاً قابلاً للنقر

### التفاصيل التقنية
- الصفحة الجديدة تجلب بيانات الشكوى من `useAdminDisputes` أو تستخدم استعلام مخصص بالـ `id`
- تعيد استخدام مكونات `DisputeFinancialImpact` و `DisputeTimeline` و `DisputeResponseThread` الموجودة
- نفس منطق الإجراءات المالية وتغيير الحالة من `DisputeCard`
- تُغلف داخل `DashboardLayout` للحفاظ على تصميم لوحة الأدمن
