## التفصيل: ما تم تنفيذه مقابل المتبقي

### أ- إدارة المنصة (Super Admin)

**تم تنفيذه:**
- إدارة جميع المستخدمين (AdminUsers)
- اعتماد أو تعليق الحسابات (is_suspended / is_verified)
- اعتماد الخدمات (AdminServices - approve/reject)
- إجراء تعديلات عبر نظام طلبات التعديل (EditRequests)
- إدارة التصنيفات والمناطق (AdminSettings)
- تحديد عمولة المنصة (CommissionForm)
- إدارة المنازعات (AdminDisputes)
- متابعة تقارير أداء المنصة (AdminReports)
- تصدير التقارير CSV (AdminReports)
- سجل التدقيق (AdminAuditLog)
- التقارير المالية (AdminFinance)
- إدارة تذاكر الدعم (AdminTickets)
- لوحة تحكم بإحصائيات شاملة (Dashboard)
- تعديل بيانات المستخدمين (UserDetailSheet)
- تسجيل جمعية أو مزود خدمة (useAdminUpdateProfile + useChangeUserRole)

**لم يتم تنفيذه:**
1. تعديل الهيدر والفوتر ومحتوى الصفحة الرئيسية من لوحة التحكم (CMS)
2. تعديل نماذج الانضمام للمنصة من لوحة التحكم
3. لوحة رحلات واضحة لكل طرف (Journey Dashboard)
4. سجل نشاط (Activity Log) لكل كيان بالتفصيل
5. التراجع عن الإجراءات الإدارية (Undo/Reverse) مع توثيق السبب
6. إخفاء/إظهار اسم الجمعية حسب إعدادات المشروع
7. إضافة سبب إلزامي عند الإيقاف/التعليق/إعادة التفعيل
8. استيراد/تصدير القوائم المرجعية (المناطق/التصنيفات)
9. حذف الخدمات أو تعليقها (تعليق مؤقت بدلا من حذف)
10. إعادة تفعيل الخدمات الموقوفة/المؤرشفة
11. إعادة تفعيل الحسابات المعلقة
12. التدخل الاستباقي قبل النزاعات
13. مراجعة سجل التعديلات لبيانات المشروع والخدمات (عرض audit_log مفلتر)

### ب- الجمعيات الشبابية

**تم تنفيذه:**
- إنشاء حساب جمعية + تسجيل دخول
- تعديل بيانات الجمعية (Profile)
- إنشاء مشاريع (ProjectCreate)
- استعراض مزودي الخدمات (Marketplace + ProviderProfile)
- اختيار مزود الخدمة (قبول عرض)
- اعتماد ساعات العمل (TimeLogs)
- شراء خدمات مصغرة (usePurchaseService)
- فتح منازعات (MyDisputes)
- الرد على المنازعات (DisputeResponseThread)
- تقييم مزودي الخدمات (Ratings)
- إرسال تذاكر للدعم (SupportTickets)
- طلبات التعديل (EditRequests)
- عرض العقود (Contracts)
- استقبال عروض (ProjectDetails)

**لم يتم تنفيذه:**
1. حفظ المشروع/الطلب كمسودة (draft) قبل الإرسال
2. التراجع عن الطلبات قبل الاعتماد
3. جعل المشروع باسم الجمعية أو بدون نشر اسمها (is_private موجود لكن لا يوجد خيار في النموذج)
4. طلب دعم مالي من المانحين لخدمة محددة

### ج- مزودو الخدمات

**تم تنفيذه:**
- إنشاء حساب + تعديل البيانات (Profile)
- إنشاء ملف مهني (Portfolio)
- تحديد سعر الساعة (hourly_rate في Profile)
- إضافة خدمات مصغرة (MyServices)
- التقديم على المشاريع (BidForm)
- تسجيل ساعات العمل (TimeTracking)
- متابعة الأرباح (Earnings)
- الرد على المنازعات (MyDisputes)
- طلب سحب الأرباح (Withdrawals)
- تقييم الجمعية (Ratings)
- طلب تعديل الخدمة (يرجع لحالة pending)
- عرض العقود والفواتير
- عرض المشاريع المتاحة (AvailableProjects)
- مشاريعي الحالية (MyProjects)

**لم يتم تنفيذه:**
1. تقديم طلب الموافقة على الملف المهني
2. حفظ الخدمة كمسودة قبل النشر
3. إيقاف الخدمة مؤقتا وإعادة تفعيلها (بدون فقد البيانات)
4. سجل الطلبات المقدمة وحالاتها بالتفصيل (Activity Log)

### د- المانحون

**تم تنفيذه:**
- إنشاء حساب مانح
- شراء خدمات (usePurchaseService)
- تخصيص الدعم لجمعيات (Donations - project/service)
- متابعة استهلاك الدعم (ImpactReports - fund consumption)
- تقارير الأثر (ImpactReports)
- عرض الجمعيات (Associations)
- التبرعات (Donations)

**لم يتم تنفيذه:**
1. رصيد الدعم وحالاته (متاح/محجوز/مستهلك/معلق/منتهي)
2. سجل تخصيصات الدعم بشكل زمني مفصل
3. الاطلاع على تقييمات كل جمعية
4. متابعة طلبات الدعم المعلقة

## خطة التنفيذ المقترحة (مرتبة حسب الأولوية)

### المرحلة 1: نظام إدارة الحالات والمرونة التشغيلية (أولوية عالية)

**الهدف:** إضافة دورة حياة مرنة للخدمات والمشاريع مع التوثيق

**التغييرات:**

1. **إضافة حالات جديدة للخدمات المصغرة**
   - إضافة حالات: `suspended` (موقوف مؤقتا), `archived` (مؤرشف), `draft` (مسودة) للـ `approval` enum
   - تحديث `MyServices` لدعم: إيقاف مؤقت / إعادة نشر / أرشفة
   - تحديث `AdminServices` لدعم: تعليق / إعادة تفعيل

2. **إضافة حالات جديدة للمشاريع**
   - إضافة حالات: `suspended`, `archived`, `pending_approval` للـ `project_status` enum
   - تحديث `ProjectCreate` لدعم حفظ كمسودة
   - تحديث `AdminProjects` لدعم: تعليق / إعادة نشر

3. **إضافة حقل السبب الإلزامي**
   - إضافة حقل `reason` عند الإيقاف/التعليق/إعادة التفعيل
   - حفظ السبب في سجل التدقيق

**الملفات المتأثرة:**
- Migration جديد لتحديث enums
- `src/pages/MyServices.tsx`
- `src/pages/admin/AdminServices.tsx`
- `src/pages/ProjectCreate.tsx`
- `src/pages/admin/AdminProjects.tsx`
- `src/hooks/useMyServices.ts`
- `src/hooks/useAdminServices.ts`

### المرحلة 2: نظام الدفع الإلكتروني وتفعيل السلة (أولوية عالية)

**الهدف:** تفعيل بوابة الدفع مع صفحة السلة وإتمام الطلب

**التغييرات:**

1. **تفعيل Stripe Integration**
   - ربط Stripe لمعالجة المدفوعات
   - إنشاء صفحة سلة الشراء (Cart)
   - صفحة إتمام الطلب (Checkout)
   - صفحة التوجيه بعد الدفع (Success/Failure)

2. **إضافة حالات مالية مفصلة**
   - تحديث `escrow_status` enum لإضافة: `pending_payment`, `failed`, `under_review`, `refunded`
   - تحديث واجهة AdminFinance لعرض الحالات الجديدة

3. **صلاحيات مالية للأدمن**
   - تعليق/استئناف الصرف
   - سجل المدفوعات والتحويلات لكل طرف

**الملفات الجديدة:**
- `src/pages/Cart.tsx`
- `src/pages/Checkout.tsx`
- `src/pages/PaymentSuccess.tsx`
- `supabase/functions/create-payment-intent/index.ts`

### المرحلة 3: إدارة المحتوى العام - CMS (أولوية عالية)

**الهدف:** تمكين السوبر أدمن من تعديل محتوى الصفحات بدون تدخل برمجي

**التغييرات:**

1. **جدول جديد `site_content`**
   - تخزين محتوى الصفحة الرئيسية (Hero, Stats, Features, Footer)
   - تخزين نصوص الهيدر والفوتر والروابط
   - تخزين سياسات المنصة والشروط والأحكام

2. **صفحة إدارة المحتوى**
   - محرر للنصوص والصور والبانرات
   - ترتيب الأقسام (drag & drop)
   - تعديل نموذج التواصل
   - تعديل قالب الفاتورة (هيدر/فوتر)

3. **تحديث الصفحة الرئيسية**
   - جلب المحتوى ديناميكيا من `site_content`

**الملفات الجديدة:**
- Migration لجدول `site_content`
- `src/pages/admin/AdminCMS.tsx`
- `src/hooks/useSiteContent.ts`
- تحديث `src/pages/Index.tsx`

### المرحلة 4: تحسين الفوترة الإلكترونية (أولوية متوسطة)

   - تعديل الهيدر/الفوتر من لوحة الأدمن
   - تحميل/إعادة إرسال الفاتورة
   - أرشفة الفواتير

**الملفات الجديدة:**
- `supabase/functions/generate-invoice-pdf/index.ts`
- تحديث `src/pages/Invoices.tsx`
- تحديث `src/pages/admin/AdminFinance.tsx`

**الهدف:** تقارير شاملة مع فلترة وتصدير متعدد الصيغ

3. **إضافة تحليلات مفقودة**
   - عدد زوار المنصة (يتطلب analytics)
   - توزيع مزودي الخدمات جغرافيا
   - تقارير المبيعات حسب الفترة والمنطقة

4. **حفظ الفلاتر المفضلة**

**الملفات المتأثرة:**

### المرحلة 6: تحسين نظام المنازعات (أولوية متوسطة)

**الهدف:** مسار حالات أكثر تفصيلا مع إرفاق الأدلة

**التغييرات:**

1. **تحديث حالات النزاع**

2. **إرفاق الأدلة والمرفقات**

3. **الأثر المالي**

## ملاحظات مهمة

- **الأولوية القصوى** هي المراحل 1-3 لأنها تمثل الفجوات الأكبر في المتطلبات الأساسية
- **نظام الدفع** يحتاج تنسيق مع بوابة الدفع (Stripe أو محلية) وهو يعتمد على قرار تجاري
- **النسخ الاحتياطي اليومي** متوفر تلقائيا عبر البنية التحتية
- **إدارة المحتوى (CMS)** هي الميزة الأكثر غيابا حاليا وتؤثر على استقلالية الأدمن عن المبرمج
- كل مرحلة مستقلة ويمكن تنفيذها بالتوازي
