

## تحسينات شاملة لتجربة مقدم الخدمة (Service Provider UX Enhancement)

بعد مراجعة جميع صفحات وتدفقات مقدم الخدمة، تم تحديد التحسينات التالية مقسمة إلى فئات:

---

### 1. لوحة التحكم (Dashboard)

| المشكلة | التحسين |
|---------|---------|
| الإحصائيات تعرض 4 بطاقات فقط وتفتقر لمعلومات حيوية | إضافة بطاقات: "عروض مقبولة"، "عقود بانتظار التوقيع"، "ساعات هذا الشهر"، "طلبات سحب معلقة" |
| لا يوجد تنبيه للعقود غير الموقعة أو العروض المقبولة | إضافة Alert بارز أعلى الصفحة للإجراءات المطلوبة (عقود بحاجة توقيع، تقييمات معلقة) |
| JourneyBoard يعرض "أرباح" كرقم خام بدون تنسيق | تنسيق المبلغ بعلامة "ر.س" وفاصلة الآلاف |
| لا يوجد قسم "نشاط أخير" خاص بالمزود | إضافة RecentActivity يعرض آخر 5 أحداث (عرض جديد، تعليق، تغيير حالة مشروع) |

### 2. خدماتي (My Services)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد فلتر حسب حالة الخدمة | إضافة شريط فلترة (الكل، معتمدة، قيد المراجعة، مسودة، موقوفة) |
| بطاقة الخدمة لا تعرض إحصائيات الأداء | إضافة عدد المشاهدات (service_views) والمبيعات (sales_count) على كل بطاقة |
| لا يوجد realtime لتحديث حالة الموافقة | إضافة اشتراك Realtime على `micro_services` لتحديث الحالة فوراً عند موافقة الأدمن |
| لا يوجد إشعار عند رفض أو موافقة الخدمة | التأكد من وجود trigger لإرسال إشعار عند تغيير approval status |
| ترتيب الخدمات ثابت حسب تاريخ الإنشاء | إضافة خيارات ترتيب (الأحدث، الأكثر مبيعاً، الأكثر مشاهدة) |

### 3. طلبات الجمعيات المتاحة (Available Projects)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد realtime لظهور طلبات جديدة | إضافة اشتراك Realtime على `projects` مع فلتر `status=open` لإظهار طلبات جديدة فوراً |
| الفلاتر لا تحفظ حالتها عند العودة للصفحة | حفظ الفلاتر في URL search params لاستمراريتها |
| عدم إظهار عدد العروض المقدمة على كل طلب | إضافة عداد العروض على بطاقة المشروع (يساعد المزود في تقييم المنافسة) |
| لا يوجد تمييز للطلبات الجديدة | إضافة شارة "جديد" للطلبات المنشورة خلال آخر 24 ساعة |

### 4. عروضي (My Bids)

| المشكلة | التحسين |
|---------|---------|
| لا يمكن تعديل العرض قبل قبوله | إضافة زر "تعديل العرض" للعروض بحالة pending مع dialog للتعديل |
| لا يوجد رابط مباشر لصفحة تفاصيل المشروع | إضافة رابط "عرض المشروع" على كل بطاقة عرض |
| لا يعرض خطاب التقديم (cover_letter) | إضافة قسم قابل للتوسيع (collapsible) لعرض خطاب التقديم والمرفقات |
| لا يظهر عدد التعليقات/المحادثات على العرض | إضافة شارة عدد التعليقات غير المقروءة |
| لا يوجد pagination | إضافة PaginationControls |

### 5. طلباتي (My Projects)

| المشكلة | التحسين |
|---------|---------|
| لا يعرض تاريخ الإسناد أو المدة المتبقية | إضافة تاريخ البدء والمدة المتوقعة وشريط تقدم الوقت |
| لا يوجد ملخص سريع للساعات والتسليمات | إضافة مؤشرات مصغرة (ساعات معتمدة/إجمالي، حالة التسليم) |
| لا يوجد pagination | إضافة PaginationControls |

### 6. المعاملات المادية (Earnings)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد realtime لتحديث حالة السحب | إضافة اشتراك Realtime على `withdrawal_requests` و`escrow_transactions` |
| لا يوجد فلتر لسجل المعاملات | إضافة فلترة حسب الحالة (محتجز، صُرف، مجمّد) وحسب التاريخ |
| لا يوجد رسم بياني للأرباح | إضافة chart بسيط يعرض الأرباح الشهرية باستخدام Recharts |
| لا يظهر مبلغ العمولة المقتطعة | إضافة عمود/سطر يوضح العمولة المقتطعة من كل معاملة |

### 7. الملف الشخصي (Profile)

| المشكلة | التحسين |
|---------|---------|
| معرض الأعمال (Portfolio) موجود لكن لا يظهر رابط مباشر من لوحة التحكم | إضافة رابط "عرض ملفي العام" في الـ header أو sidebar |
| لا يوجد معاينة فورية للملف الشخصي العام | إضافة زر "معاينة الملف العام" يفتح صفحة `/profile/:id` |

### 8. تحسينات Realtime والإشعارات المفقودة

| السياق | التحسين |
|---------|---------|
| خدماتي لا تستخدم Realtime | إضافة اشتراك Realtime على `micro_services` بفلتر `provider_id` |
| Available Projects لا تستخدم Realtime | إضافة اشتراك لإظهار طلبات جديدة فور نشرها |
| Earnings لا تستخدم Realtime | إضافة اشتراك على `escrow_transactions` و `withdrawal_requests` |
| لا يوجد إشعار صوتي أو مرئي للطلبات الجديدة | إضافة toast تلقائي عند ظهور طلب جديد مطابق لمهارات المزود |

### 9. تحسينات UX عامة

| المشكلة | التحسين |
|---------|---------|
| صفحة تقديم العرض (ProjectBidView) لا تعرض العروض الأخرى أو متوسط السعر | إضافة معلومات إحصائية (عدد العروض، متوسط السعر المقدم) لمساعدة المزود في التسعير |
| لا يوجد onboarding أو checklist للمزود الجديد | إضافة خطوات بدء (أكمل ملفك، أضف أول خدمة، قدم أول عرض) |
| عدم وجود صفحة "لمحة عامة" تجمع كل المؤشرات | Dashboard الحالي جيد لكن يمكن إضافة قسم "مهام مطلوبة" (Action Items) |

---

### أولوية التنفيذ المقترحة

1. **Realtime + إشعارات** — خدماتي، المعاملات، الطلبات المتاحة
2. **Dashboard Action Items** — تنبيهات العقود والتقييمات والإجراءات المطلوبة
3. **My Bids تحسينات** — تعديل العرض، عرض التفاصيل، pagination
4. **My Services فلترة وإحصائيات** — فلتر الحالة، عداد المشاهدات/المبيعات
5. **Earnings رسم بياني وفلترة** — chart شهري، فلتر المعاملات
6. **Available Projects تحسينات** — شارة "جديد"، عداد العروض، حفظ الفلاتر
7. **My Projects مؤشرات** — تقدم الوقت والساعات
8. **Onboarding checklist** — للمزودين الجدد

### التفاصيل التقنية

- Realtime subscriptions تستخدم نفس النمط الموجود في `useMyAssignedProjects` و `useProviderBids`
- Charts تستخدم `recharts` (مثبت بالفعل)
- URL search params عبر `useSearchParams` من react-router-dom
- جميع التعديلات تحترم RLS policies الموجودة ولا تحتاج migrations جديدة
