

## مراجعة شاملة للمصادقة والصفحة الرئيسية (Authentication & Landing Page UX Enhancement)

---

### 1. المصادقة (AuthModal)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد مؤشر قوة كلمة المرور | إضافة شريط تقدم يوضح قوة كلمة المرور (ضعيفة/متوسطة/قوية) أثناء التسجيل |
| لا يوجد خيار إظهار/إخفاء كلمة المرور | إضافة زر eye/eye-off toggle لحقل كلمة المرور |
| بعد تسجيل الدخول الناجح لا يوجد toast ترحيبي | إضافة toast "مرحباً [الاسم]" بعد تسجيل الدخول |
| نموذج التسجيل طويل ويظهر دفعة واحدة | تقسيم التسجيل لخطوتين: (1) الاسم + الدور + الجوال، (2) البريد + كلمة المرور + الموافقة — مع مؤشر StepProgress |
| لا يتم إعادة توجيه المستخدم المسجّل تلقائياً عند فتح `/auth` | إضافة redirect تلقائي من `/auth` إلى `/dashboard` إذا كان المستخدم مسجّل دخوله |
| رابط "نسيت كلمة المرور" يفتح صفحة جديدة ويغلق المودال | فتح "نسيت كلمة المرور" بتدفق مدمج داخل المودال أو التأكد من إغلاقه بشكل نظيف |

### 2. استعادة كلمة المرور (ForgotPassword + ResetPassword)

| المشكلة | التحسين |
|---------|---------|
| صفحة ForgotPassword لا تتبع تصميم PublicLayout (بدون header/footer) | لف ForgotPassword وResetPassword بـ PublicLayout أو إضافة رابط عودة واضح مع الشعار |
| ResetPassword لا تعرض رسالة واضحة عند فشل التحقق (مثلاً رابط منتهي) | إضافة حالة خطأ واضحة مع زر "أعد الطلب" يوجه لـ `/forgot-password` |
| لا يوجد مؤشر قوة كلمة المرور في ResetPassword | إضافة نفس مؤشر القوة المستخدم في التسجيل |

### 3. الصفحة الرئيسية — Hero Section

| المشكلة | التحسين |
|---------|---------|
| زر "ابدأ الآن" يوجه دائماً لـ `/auth` حتى للمستخدم المسجّل | إذا كان المستخدم مسجّل: تغيير النص لـ "لوحة التحكم" والتوجيه لـ `/dashboard` |
| لا يوجد زر CTA ثانوي في Hero | إضافة زر "تصفح الخدمات" كزر ثانوي يوجه لقسم الخدمات أو `/marketplace` |

### 4. الصفحة الرئيسية — طلبات الجمعيات (LandingRequestsTable)

| المشكلة | التحسين |
|---------|---------|
| زر CTA السفلي يوجه دائماً لصفحة التسجيل حتى للمسجّلين | إذا كان المستخدم مسجّل: توجيه لـ `/available-projects` بدلاً من `/auth` |
| لا يظهر عدد العروض المقدمة على كل طلب | إضافة badge "X عروض" لكل طلب (يتطلب إضافة count في الـ query) |

### 5. الصفحة الرئيسية — الخدمات (LandingServicesGrid)

| المشكلة | التحسين |
|---------|---------|
| زر CTA السفلي يوجه لـ `/auth` حتى للمسجّلين | إذا كان المستخدم مسجّل: توجيه لـ `/marketplace` |
| لا يظهر عدد المبيعات أو التقييم على بطاقة الخدمة | إضافة عداد مبيعات (sales_count) وتقييم متوسط على ServiceCard |
| لا يوجد شارة "مميزة" على الخدمات المميزة (is_featured) | إضافة شارة ⭐ "مميزة" على بطاقات الخدمات ذات `is_featured=true` |

### 6. الصفحة الرئيسية — الإحصائيات (LiveStats)

### 10. صفحات عامة — تفاصيل الخدمة والطلب (ServiceDetail, ProjectPublicView)

| لا يوجد مؤشر للصفحة النشطة (active link) | إضافة تمييز بصري للرابط النشط |

### أولوية التنفيذ

1. **Auth redirect + Hero CTA ذكي** — توجيه المستخدم المسجّل لـ Dashboard بدلاً من Auth
2. **إظهار/إخفاء كلمة المرور + مؤشر القوة** — تحسين UX الأساسي للمصادقة
3. **CTAs ذكية** — تحديث أزرار CTA في Requests/Services لتكون سياقية (مسجّل/زائر)
4. **التسجيل بخطوتين** — تقسيم نموذج التسجيل لخطوتين مع StepProgress
5. **شارات الخدمات** — مميزة، عداد مبيعات، تقييم
6. **Active link + breadcrumbs** — تحسينات تنقل
7. **ContactForm ملء تلقائي** — ملء البريد للمسجّلين
8. **ForgotPassword/ResetPassword** — تحسين التصميم ورسائل الخطأ

### التفاصيل التقنية

- مؤشر قوة كلمة المرور: مكوّن `PasswordStrength` يحسب القوة بناءً على الطول، الأحرف الكبيرة/الصغيرة، الأرقام، الرموز
- التسجيل بخطوتين: استخدام `useState` لإدارة الخطوة الحالية + `StepProgress` الموجود
- CTAs الذكية: استخدام `useAuth()` للتحقق من حالة تسجيل الدخول وتغيير النص والوجهة
- شارات الخدمات: البيانات موجودة (`is_featured`, `sales_count`) — فقط عرضها في البطاقة
- Active link: استخدام `useLocation()` مع مقارنة `pathname`
- لا حاجة لـ migrations جديدة (باستثناء تحديث `get_landing_stats` لإضافة عدد المانحين)
