

## تحسينات شاملة لتجربة الجمعية (Youth Association UX Enhancement)

بعد مراجعة جميع صفحات وتدفقات الجمعية الشبابية:

---

### 1. لوحة التحكم (Dashboard)

| المشكلة | التحسين |
|---------|---------|
| الإحصائيات تفتقر لمعلومات حيوية (ساعات معلقة، عقود غير موقعة) | إضافة بطاقات: "ساعات بانتظار الموافقة"، "عقود بحاجة لتوقيع"، "طلبات منح معلقة" |
| لا يوجد تنبيه للعقود غير الموقعة أو التسليمات المعلقة | إضافة Alert بارز: عقود بحاجة توقيع، تسليمات بانتظار المراجعة، تقييمات معلقة |
| لا يوجد قسم "إجراءات مطلوبة" (Action Items) | إضافة AssociationActionItems مشابه لـ ProviderActionItems الموجود |
| JourneyBoard لا يعرض رصيد المنح أو المبالغ المصروفة | إضافة خطوة "المنح" مع رصيد المنح المتاح |

### 2. طلبات الجمعيات (Projects)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد عداد للعروض المقدمة على كل طلب | إضافة badge عدد العروض على ProjectCard (البيانات موجودة: bids relation) |
| لا يوجد شريط تقدم يوضح مرحلة الطلب | إضافة مؤشر مرئي للمرحلة (مسودة → مراجعة → مفتوح → تنفيذ → مكتمل) |
| لا يوجد pagination للطلبات | إضافة PaginationControls |
| لا يوجد بحث نصي في الطلبات | إضافة حقل بحث بالعنوان |

### 3. تفاصيل الطلب (ProjectDetails)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد ملخص مالي واضح في أعلى الصفحة | إضافة بطاقة ملخص مالي (ميزانية، ضمان محتجز، عمولة، المبلغ الصافي) |
| ساعات العمل لا تعرض ملخصاً سريعاً | إضافة شريط تقدم الساعات (معتمدة / معلقة / إجمالي مقدّر) |
| لا يوجد realtime للتعليقات على العروض (bid_comments) | إضافة اشتراك Realtime على bid_comments |
| لا يوجد realtime لسجلات الوقت والتسليمات | إضافة اشتراك Realtime على time_logs و project_deliverables |

### 4. سوق الخدمات (Marketplace)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد تمييز للخدمات المميزة (is_featured) | إضافة شارة "مميزة" على بطاقات الخدمات المميزة |
| لا يوجد عداد مبيعات على بطاقة الخدمة | إضافة عدد المبيعات (sales_count) والتقييم على ServiceCard |
| الفلاتر لا تحفظ حالتها في URL | حفظ الفلاتر في URL search params |

### 5. سلة المشتريات والدفع (Cart + Checkout)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد تنبيه عند توفر رصيد منح كافٍ للدفع | إضافة Alert يوضح رصيد المنح المتاح ويقترح الدفع منه |
| لا يظهر ملخص رصيد المنح في صفحة السلة | عرض رصيد المنح المتاح قبل زر "إتمام الشراء" |

### 6. المنح المستلمة (ReceivedGrants)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد فلتر حسب حالة المنحة أو المانح | إضافة فلترة حسب الحالة (متاح، محجوز، مستهلك) وحسب المانح |
| لا يوجد pagination | إضافة PaginationControls |
| لا يوجد realtime لتحديث المنح الجديدة | إضافة اشتراك Realtime على donor_contributions |
| الصفحة تفتقر للـ page header المعياري | إضافة header وdivider متسق مع باقي الصفحات |

### 7. طلبات المنح (MyGrants)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد فلتر حسب حالة الطلب | إضافة شريط فلترة (الكل، معلقة، تمت الموافقة، ممولة، مرفوضة) |
| لا يوجد pagination | إضافة PaginationControls |
| لا يوجد realtime لتحديث حالة الطلب | إضافة اشتراك Realtime على grant_requests |
| لا يعرض معلومات عن الميزانية المتبقية للمشروع المرتبط | إضافة badge بالميزانية والمبلغ المتبقي عند ربط الطلب بمشروع |

### 8. تقارير الأثر (AssociationImpactReports)

| المشكلة | التحسين |
|---------|---------|
| الصفحة تفتقر للـ page header المعياري والـ gradient divider | إضافة header متسق |
| لا يمكن ربط التقرير بمنحة محددة (contribution_id) | إضافة خيار تحديد المنحة المرتبطة في نموذج الرفع |
| لا يوجد فلتر حسب المانح | إضافة فلترة حسب المانح |

### 9. العقود (Contracts)

| المشكلة | التحسين |
|---------|---------|
| لا يوجد realtime لتحديث حالة التوقيع | إضافة اشتراك Realtime على contracts (موجود بالفعل في ProjectDetails لكن ليس في صفحة العقود المستقلة) |
| لا يوجد pagination | إضافة PaginationControls |

### 10. تحسينات Realtime والإشعارات المفقودة

| السياق | التحسين |
|---------|---------|
| المنح المستلمة لا تستخدم Realtime | إضافة اشتراك على donor_contributions بفلتر association_id |
| طلبات المنح لا تستخدم Realtime | إضافة اشتراك على grant_requests بفلتر association_id |
| العقود المستقلة لا تستخدم Realtime | إضافة اشتراك على contracts بفلتر association_id |
| لا يوجد toast تلقائي عند تسليم ملفات من المزود | إضافة toast فوري عند INSERT في project_deliverables |

### 11. تحسينات UX عامة

| المشكلة | التحسين |
|---------|---------|
| لا يوجد onboarding checklist للجمعية الجديدة | إضافة خطوات بدء (أكمل ملف الجمعية، أنشئ أول طلب، أو اشترِ خدمة) |
| لا يوجد معاينة فورية للملف الشخصي العام | إضافة زر "معاينة الملف العام" في الملف الشخصي |
| صفحة المانحين (Donors) لا تعرض إجمالي المنح المستلمة من كل مانح | إضافة badge بإجمالي المنح السابقة من هذا المانح |

---

### أولوية التنفيذ المقترحة

1. **Dashboard Action Items** — تنبيهات العقود والتسليمات وساعات العمل المعلقة
2. **Realtime + إشعارات** — المنح، طلبات المنح، العقود، التسليمات
3. **Projects تحسينات** — عداد العروض، شريط التقدم، بحث، pagination
4. **ProjectDetails ملخص مالي** — بطاقة مالية وشريط تقدم الساعات
5. **ReceivedGrants فلترة وRealtime** — فلتر حالة، header متسق
6. **MyGrants فلترة وRealtime** — فلتر حالة، pagination
7. **Marketplace تحسينات** — خدمات مميزة، حفظ فلاتر URL
8. **Cart رصيد المنح** — تنبيه رصيد المنح المتاح
9. **Impact Reports تحسينات** — header، ربط بمنحة، فلتر
10. **Onboarding checklist** — للجمعيات الجديدة

### التفاصيل التقنية

- Realtime subscriptions تستخدم نفس النمط الموجود في `useProjects` (خطاف channel + invalidateQueries)
- بطاقات العروض تستخدم البيانات الموجودة بالفعل (bids relation في useProjects)
- لا حاجة لـ migrations جديدة — جميع البيانات المطلوبة متوفرة في الجداول الحالية
- PaginationControls ومكوّن usePagination موجودان بالفعل وجاهزان للاستخدام
