

# تقييد خيارات حالة الموافقة حسب الحالة الحالية للخدمة

## المشكلة
القائمة المنسدلة تعرض جميع الحالات (مسودة، قيد المراجعة، مقبول، مرفوض، موقوف، مؤرشف) بغض النظر عن الحالة الحالية — وهذا يخالف الاتفاق السابق بأن الأدمن بعد الموافقة يقدر فقط يعلّق الخدمة.

## الحل
تقييد الخيارات المتاحة في `Select` بناءً على الحالة الحالية:

| الحالة الحالية | الخيارات المتاحة |
|---|---|
| `pending` | مقبول، مرفوض |
| `approved` | موقوف |
| `suspended` | مقبول (إعادة تفعيل) |
| `draft` / `rejected` | لا خيارات — القائمة معطلة |

## التغييرات

### `src/pages/admin/AdminServiceDetail.tsx` (سطور 156-168)
- إضافة دالة `getAllowedTransitions(currentApproval)` تُرجع قائمة الحالات المسموحة
- إذا كانت القائمة فارغة → يُعطّل الـ `Select` بالكامل (`disabled`)
- عرض فقط الخيارات المسموحة داخل `SelectContent`
