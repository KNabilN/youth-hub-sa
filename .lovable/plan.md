

# خطة: تحكم كامل في محتوى الصفحة الرئيسية عبر CMS

## الوضع الحالي
الأقسام المُدارة عبر CMS: Hero ✓، المميزات ✓، لماذا المنصة ✓، آراء العملاء ✓
الأقسام **غير المُدارة** (نصوص ثابتة في الكود):

| القسم | النصوص الثابتة |
|---|---|
| طلبات الجمعيات | العنوان، الوصف، نص الزر |
| الخدمات المتوفرة | العنوان، الوصف، نص الزر |
| CTA النهائي | النصوص ثابتة رغم وجود بيانات CMS |
| قسم التواصل | العنوان والوصف ثابتان |

بالإضافة لذلك: لا يوجد تحكم في **إظهار/إخفاء** الأقسام أو تغيير صورة خلفية CTA.

---

## التغييرات المطلوبة

### 1. إضافة أقسام CMS جديدة (Migration)
إضافة 3 صفوف جديدة في `site_content`:
- `requests_section` → `{ title, subtitle, button_text, visible }`
- `services_section` → `{ title, subtitle, button_text, visible }`
- `contact_section` → `{ title, subtitle, visible }`

وإضافة حقل `visible` لكل الأقسام الحالية (hero, features, trust, testimonials, cta).

### 2. تحديث Index.tsx
- قسم **طلبات الجمعيات**: قراءة العنوان/الوصف/نص الزر من `requests_section`
- قسم **الخدمات**: قراءة العنوان/الوصف/نص الزر من `services_section`
- قسم **CTA**: استخدام بيانات CMS بدلاً من النص الثابت
- قسم **التواصل**: قراءة العنوان/الوصف من `contact_section`
- إضافة دعم `visible` لكل قسم (إخفاء القسم عند `visible: false`)

### 3. تحديث LandingRequestsTable + LandingServicesGrid
تمرير العناوين والنصوص كـ props بدلاً من الثوابت.

### 4. تحديث AdminCMS.tsx
إضافة محررات للأقسام الجديدة مع:
- حقول النصوص (عنوان، وصف، نص الزر)
- مفتاح تبديل (Switch) لإظهار/إخفاء كل قسم
- إضافة محرر صورة خلفية لقسم CTA
