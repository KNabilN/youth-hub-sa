

# خطة تنفيذ: نظام رفع الملفات والمرفقات

## ملخص
إضافة نظام متكامل لرفع الملفات وإرفاقها بالمشاريع والعقود وتذاكر الدعم، مع واجهة سهلة الاستخدام تدعم السحب والإفلات.

---

## المراحل

### المرحلة 1: إعداد قاعدة البيانات والتخزين

**إنشاء Storage Bucket:**
- bucket باسم `attachments` (خاص - private)
- سياسات تخزين تسمح للمستخدمين المسجلين برفع وعرض ملفاتهم

**إنشاء جدول `attachments`:**

| العمود | النوع | الوصف |
|--------|------|-------|
| id | UUID | المعرف الأساسي |
| user_id | UUID | من رفع الملف |
| entity_type | TEXT | نوع الكيان (project / contract / ticket) |
| entity_id | UUID | معرف الكيان المرتبط |
| file_name | TEXT | اسم الملف الأصلي |
| file_path | TEXT | المسار في التخزين |
| file_size | INTEGER | حجم الملف بالبايت |
| mime_type | TEXT | نوع الملف |
| created_at | TIMESTAMPTZ | تاريخ الرفع |

**سياسات الأمان (RLS):**
- المستخدمون يرون مرفقات الكيانات التي لديهم صلاحية الوصول إليها
- المستخدمون يرفعون مرفقات فقط للكيانات المرتبطة بهم
- صاحب المرفق يمكنه حذفه
- المدير لديه صلاحية كاملة

---

### المرحلة 2: مكونات واجهة المستخدم

**1. مكون `FileUploader` (قابل لإعادة الاستخدام):**
- منطقة سحب وإفلات مع زر اختيار ملف
- عرض شريط تقدم أثناء الرفع
- التحقق من نوع الملف والحجم (10 MB كحد أقصى)
- الأنواع المسموحة: PDF, PNG, JPG, JPEG, WEBP, DOC, DOCX, XLS, XLSX
- يستقبل `entityType` و `entityId` كخصائص

**2. مكون `AttachmentList` (عرض المرفقات):**
- قائمة المرفقات مع أيقونة نوع الملف واسمه وحجمه
- زر تحميل لكل ملف
- زر حذف (لصاحب المرفق فقط)
- حالة فارغة عند عدم وجود مرفقات

---

### المرحلة 3: Hook مخصص

**`useAttachments` hook:**
- `useAttachments(entityType, entityId)` - جلب المرفقات
- `useUploadAttachment()` - رفع ملف جديد
- `useDeleteAttachment()` - حذف مرفق

---

### المرحلة 4: دمج المرفقات في الصفحات

**1. صفحة تفاصيل المشروع (`ProjectDetails.tsx`):**
- إضافة تبويب جديد "المرفقات" في نظام التبويبات الموجود
- عرض `FileUploader` + `AttachmentList`

**2. تبويب العقد (في نفس الصفحة):**
- إضافة قسم مرفقات أسفل تفاصيل العقد

**3. تذاكر الدعم:**
- إضافة إمكانية إرفاق ملفات عند إنشاء تذكرة جديدة في `TicketCreate.tsx`

---

## التفاصيل التقنية

### هيكل الملفات الجديدة
```text
src/
  hooks/
    useAttachments.ts          -- hook للتعامل مع المرفقات
  components/
    attachments/
      FileUploader.tsx         -- مكون رفع الملفات
      AttachmentList.tsx        -- مكون عرض المرفقات
```

### مسار التخزين
```text
attachments/{user_id}/{entity_type}/{entity_id}/{timestamp}_{filename}
```

### قيود الملفات
- الحجم الأقصى: 10 ميجابايت
- الأنواع: PDF, PNG, JPG, JPEG, WEBP, DOC, DOCX, XLS, XLSX

### سياسات Storage RLS
- INSERT: المستخدم المسجل يرفع في مجلده الخاص
- SELECT: المستخدمون المسجلون يمكنهم تحميل الملفات
- DELETE: صاحب الملف فقط (بناء على المسار)

### سياسات جدول attachments (RLS)
- SELECT: المستخدم يرى مرفقات المشاريع/العقود/التذاكر المرتبطة به + المدير يرى الكل
- INSERT: المستخدم يضيف مرفقات للكيانات المرتبطة به + user_id = auth.uid()
- DELETE: صاحب المرفق (user_id = auth.uid()) + المدير
