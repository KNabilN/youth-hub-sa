

## خطة: إصلاح عرض بروفايل الجمعيات وزر الدعم

### المشكلة الجذرية

سياسة RLS على جدول `user_roles` تسمح فقط للمستخدم بقراءة دوره الخاص. عندما يفتح المانح بروفايل جمعية، لا يستطيع قراءة دور الجمعية → `role.data = null` → `isAssociation = false` → تظهر تبويبات مقدم الخدمة بدلاً من تبويبات الجمعية ولا يظهر زر "دعم هذه الجمعية".

### الحل

إضافة سياسة RLS جديدة تسمح لأي مستخدم مسجّل بقراءة أدوار المستخدمين الآخرين (الدور معلومة غير حساسة وتُعرض في البروفايل العام أصلاً).

### التغييرات

#### 1. Migration: إضافة سياسة RLS جديدة
```sql
CREATE POLICY "Authenticated users can view all roles"
ON public.user_roles
FOR SELECT
TO authenticated
USING (true);
```
هذا سيحل محل السياسة الحالية "Users can view own role" ويسمح لأي مستخدم مسجّل بمعرفة دور أي مستخدم آخر.

#### 2. لا تغييرات في الكود
الكود الحالي في `PublicProfile.tsx` و `usePublicProfile.ts` يعمل بشكل صحيح — المشكلة فقط في الصلاحيات.

### النتيجة المتوقعة
- المانح يرى زر "دعم هذه الجمعية" في بروفايل أي جمعية
- الجمعيات تعرض تبويبات (نبذة، الطلبات، التقييمات) بدلاً من (الخدمات، الأعمال)

| الملف | التغيير |
|-------|---------|
| Migration SQL | إضافة سياسة RLS للسماح بقراءة الأدوار |
