

# إصلاح الخط العربي وتحسين تصميم التقرير الحكومي

## المشكلة
النص العربي في تقرير PDF يظهر معكوساً/مشوهاً (كما في الصورة) بسبب مشكلة معروفة في `html2canvas` مع نصوص RTL. بالإضافة لذلك، التصميم يحتاج ترقية ليكون بمستوى التقارير الحكومية الرسمية.

## الحل

### 1. إصلاح الخط العربي (`src/lib/pdf-utils.ts`)
- تحميل عدة أوزان من خط Cairo (Regular + Bold + ExtraBold) لضمان التغطية الكاملة
- إضافة خط Noto Sans Arabic كبديل احتياطي
- إضافة علامات Unicode bidi (`\u200F` RLM + `\u2067` RLI) لمنع انعكاس النص

### 2. إصلاح الحاوية المخفية (`src/lib/report-pdf.ts`)
- إضافة `unicode-bidi: bidi-override` و `text-align: right` للحاوية الرئيسية
- لف كل نص عربي بـ `<bdi dir="rtl">` لعزل اتجاه النص
- إضافة helper function `bidi(text)` يلف النصوص العربية بعلامات Unicode المناسبة
- زيادة `scale` من 2 إلى 3 لجودة أعلى في الطباعة

### 3. تحسين تصميم التقرير للمستوى الحكومي
- **هيدر رسمي**: إضافة شريط ذهبي علوي (accent bar)، عنوان "وثيقة رسمية"، ورقم مرجعي فريد
- **بطاقات الإحصائيات**: تصميم أكبر مع حدود مزدوجة وظلال خفيفة، وعرض 4 بطاقات في الصف بدلاً من عرض الكل في صف واحد
- **الجداول**: إضافة ترقيم تلقائي للصفوف، خطوط فاصلة أوضح، وأحجام خط أكبر
- **فوتر رسمي**: إضافة نص "وثيقة سرية" وختم المنصة مع رقم الصفحة
- **ترقيم الصفحات**: إضافة أرقام صفحات تلقائية في أسفل كل صفحة

### 4. إضافة صفحة غلاف (Cover Page)
- صفحة كاملة كغلاف تتضمن: الشعار الكبير، عنوان التقرير، الفترة الزمنية، اسم المنصة، وتاريخ الإصدار

### الملفات المتأثرة:
- `src/lib/pdf-utils.ts` — تحميل خطوط إضافية + helper للـ bidi
- `src/lib/report-pdf.ts` — إصلاح RTL + تصميم حكومي محسّن
