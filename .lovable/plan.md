
# تخصيص ملف المانح الشخصي + رفع شعار الشركة

## المشكلة
حالياً صفحة الملف الشخصي للمانح تعرض نفس أقسام مقدم الخدمة (المهارات، المؤهلات والشهادات، النبذة التعريفية بنفس الشكل) وهي غير مناسبة لدور المانح. المانح يحتاج معلومات مختلفة تتعلق بجهته التمويلية.

## التغييرات المطلوبة

### 1. تغييرات قاعدة البيانات
إضافة عمود `company_logo_url` لجدول `profiles`:
```sql
ALTER TABLE profiles ADD COLUMN company_logo_url text DEFAULT '';
```

### 2. تعديل صفحة الملف الشخصي (`src/pages/Profile.tsx`)

**إخفاء الأقسام غير المناسبة للمانح:**
- اخفاء قسم "المهارات" (Skills) عن المانح
- اخفاء قسم "المؤهلات والشهادات" (Qualifications) عن المانح
- اخفاء قسم "معرض الأعمال" (PortfolioManager) - هذا حالياً فقط لمقدم الخدمة وهو صحيح

**إضافة أقسام خاصة بالمانح:**

**أ. قسم "معلومات الجهة المانحة"** (بدلاً من المعلومات الأساسية العامة):
- الاسم الكامل (مطلوب)
- رقم الهاتف (مطلوب)
- اسم المنظمة/الشركة (اختياري - استخدام حقل `organization_name` الموجود)
- النبذة التعريفية: تغيير العنوان الفرعي ليصبح "اكتب نبذة عن جهتك المانحة وأهدافك في الدعم"

**ب. قسم "شعار الجهة المانحة":**
- منطقة رفع شعار الشركة/الجهة (صورة)
- يُخزن في حاوية `avatars` بمسار `{userId}/company-logo.{ext}`
- يُحفظ الرابط في حقل `company_logo_url` الجديد
- عرض الشعار الحالي مع إمكانية تغييره
- تصميم بطاقة مع أيقونة `Building` ومنطقة drag-and-drop/click-to-upload

### 3. Hook رفع شعار الشركة (`src/hooks/useUploadCompanyLogo.ts` - جديد)
- يشبه `useUploadAvatar` لكن يرفع الملف بمسار `{userId}/company-logo.{ext}`
- يحدّث حقل `company_logo_url` في `profiles`
- يُبطل cache الـ profile عند النجاح

### 4. تعديل شروط إظهار الأقسام في Profile.tsx

| القسم | مقدم خدمة | جمعية | مانح | مدير |
|---|---|---|---|---|
| المعلومات الأساسية | نعم | نعم | نعم (مخصص) | نعم |
| النبذة التعريفية | نعم | نعم | نعم (عنوان مخصص) | نعم |
| شعار الجهة المانحة | لا | لا | نعم | لا |
| اسم المنظمة | لا | نعم | نعم (اختياري) | لا |
| البيانات البنكية | نعم | نعم | لا | لا |
| المهارات | نعم | نعم | لا | لا |
| المؤهلات | نعم | نعم | لا | لا |
| معرض الأعمال | نعم | لا | لا | لا |
| ضابط الاتصال | لا | نعم | لا | لا |

### 5. تعديل `useProfileCompleteness.ts`
إضافة حقول اختيارية للمانح (لا تؤثر على نسبة الاكتمال لأنها اختيارية):
- لا تغيير - الحقول المطلوبة للمانح تبقى فقط `full_name` و `phone`

## الملفات المتأثرة

| الملف | التغيير |
