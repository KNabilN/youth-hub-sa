

# خطة: إضافة نسبة المنصة وضريبة القيمة المضافة (15%) فوق المبلغ الأساسي

## الوضع الحالي
- المبلغ المعروض والمدفوع = سعر الخدمة فقط (مثلاً 1000 ر.س)
- نسبة المنصة تُحسب داخلياً عند إصدار الفاتورة وتُخصم من المبلغ
- لا يوجد ذكر لضريبة القيمة المضافة في أي مكان

## المطلوب
عند الدفع، يظهر للمستخدم:
```text
سعر الخدمة:          1,000 ر.س
رسوم المنصة (5%):       50 ر.س
ضريبة القيمة المضافة (15%):  7.50 ر.س   ← على رسوم المنصة
─────────────────────────────
الإجمالي:            1,057.50 ر.س
```

## التغييرات المطلوبة

### 1. دالة حساب موحدة (ملف جديد)
**ملف:** `src/lib/pricing.ts`
- دالة `calculatePricing(baseAmount, commissionRate)` تُرجع:
  - `subtotal` = المبلغ الأساسي
  - `commission` = المبلغ × نسبة المنصة
  - `vat` = العمولة × 0.15
  - `total` = subtotal + commission + vat
- دالة `useCommissionRate()` hook لجلب النسبة من `commission_config`

### 2. صفحة السلة (`Cart.tsx`)
- عرض تفصيل الأسعار: المجموع الفرعي + رسوم المنصة + الضريبة + الإجمالي
- يحتاج جلب نسبة العمولة

### 3. صفحة الدفع (`Checkout.tsx`)
- عرض التفصيل في ملخص الدفع بدل المبلغ الواحد
- تمرير `total` الجديد (شامل العمولة والضريبة) إلى Moyasar وللتحويل البنكي
- تحديث `paymentContext.total` ليشمل المبلغ الكامل
- تحديث المبلغ المعروض في بيانات الحساب البنكي

### 4. صفحة المنح (`Donations.tsx` + `DonationPaymentStep.tsx`)
- نفس المنطق: إضافة العمولة والضريبة فوق مبلغ المنحة
- عرض التفصيل قبل الدفع
- تمرير المبلغ الكامل لـ Moyasar والتحويل البنكي

### 5. Edge Function (`moyasar-verify-payment`)
- المبلغ القادم من Moyasar سيكون المبلغ الكامل (شامل العمولة والضريبة)
- عند إنشاء escrow: يُحفظ المبلغ الأساسي فقط (سعر الخدمة)
- العمولة والضريبة تُسجل في الفاتورة

### 6. التحويل البنكي (`useBankTransfer.ts`)
- المبلغ المرسل للبنك = المبلغ الكامل
- escrow يُسجل بالمبلغ الأساسي
- الفاتورة تتضمن تفاصيل العمولة والضريبة

### 7. صفحة النجاح (`PaymentSuccess.tsx`)
- عرض المبلغ الإجمالي المدفوع (شامل الرسوم)
